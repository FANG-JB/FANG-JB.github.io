设计模式(1)-单例模式

java java 设计模式 2021/07/22

通常某个类不需要重复的实例化对象，只需要在系统中实例一次即可，就可以用到单例模式。

饿汉式

实现步骤

  1. 私有化构造器
  2. 实例化一个静态对象实例
  3. 对外提供一个静态方法返回实例对象

代码

优点：避免了线程同步问题

缺点：提前实例化，可能造成内存的不必要占用。

懒汉式

实现步骤

  1. 私有化构造器
  2. 声明一个私有对象
  3. 对外提供一个方法，当有外部调用实例方法时，才进行对象的实例

代码

优点：实现懒加载实例化，避免内存的不必要占用

缺点：在多线程的情况下是不安全的

改进

在方法上增加synchronized保证线程安全

缺点：代码效率降低

双重检查

线程安全和效率都可以

静态内部类

静态内部类在被调用的时候才会被装载，实现了懒加载，并且避免了线程不安全

枚举

最简单的实现方式， 推荐使用。
